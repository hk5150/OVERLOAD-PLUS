# OVERLOAD+

プログレッシブオーバーロードに基づいて、次回のトレーニングメニュー(重量・回数・セット数)を自動で提案する記録アプリ。

判定はルールエンジン(コード)が決定論的に行うため、毎回ブレません。

---

## 主な機能

- **記録**: 重量 / 回数 / RIR(あと何回できたか)、ウォームアップとワーキングセットの区別
- **自動メニュー**: 分割のローテーションに応じて、その日のメニューを次回ぶんの重量・回数に調整して提示
- **ルールエンジン**: ダブルプログレッション。全セット上限達成→増量、未達→回数+1、停滞→デロード、痛み→増量停止 など
- **分割法**: 全身 / 上下 / PPL / 5分割のプリセット、カスタム分割、Dayごとの種目登録
- **種目マスター**: 約95種目を内蔵。カスタム種目の追加・永続保存
- **推定1RM・PR**: Epley式による推定1RMとPR(自己ベスト)、PR更新バッジ
- **グラフ**: 総ボリューム推移(トータル/部位別)、種目別の重量・推定1RM推移
- **タイマー**: セッション経過時間、インターバル(記録入力後に自動スタート)
- **部位別集計**: 直近7日のセット数とボリューム、最終実施日
- **可動域係数**: シュラッグ・カーフレイズなど可動域の小さい種目は集計ボリュームを0.5倍で換算(1RM・判定には影響しない)
- **バックアップ**: JSON書き出し / 復元、CSV書き出し
- **オフライン対応**: Service Workerにより、一度開けばオフラインでも起動可能

## GitHub Pages で公開する手順

### 1. リポジトリを作る

1. GitHub にログインし、右上の **+** → **New repository**
2. **Repository name** に `overload` などを入力
3. **Public** を選択(Pages を無料で使うため)
4. **Create repository** をクリック

### 2. ファイルをアップロードする

1. できたリポジトリの画面で **Add file** → **Upload files**
2. 次の **7ファイルすべて** をドラッグ&ドロップ

   ```
   index.html
   manifest.json
   sw.js
   icon-192.png
   icon-512.png
   icon-maskable-512.png
   apple-touch-icon.png
   ```

   (この `README.md` も一緒に上げてよい)

3. 下部の **Commit changes** をクリック

### 3. Pages を有効にする

1. リポジトリの **Settings** タブ
2. 左メニューの **Pages**
3. **Source** を **Deploy from a branch** に設定
4. **Branch** を **main** / **/(root)** にして **Save**
5. 1〜2分待つと、ページ上部に公開URLが表示される

   ```
   https://<ユーザー名>.github.io/overload/
   ```

### 4. iPhone のホーム画面に追加する

1. **Safari** で上記URLを開く(Chrome ではなく Safari)
2. 下部の **共有ボタン**(□に↑)をタップ
3. 下にスクロールして **ホーム画面に追加**
4. **追加** をタップ

ホーム画面のアイコンから、全画面のアプリとして起動します。

### Android の場合

Chrome でURLを開き、メニュー(⋮)→ **ホーム画面に追加** / **アプリをインストール**

## データについて

- データは **使っている端末のブラウザ内(localStorage)** に保存されます。サーバーには何も送信されません。
- **端末やブラウザが変わるとデータは引き継がれません。**
- **重要**: 定期的に「設定」タブから **バックアップを書き出す** ことを推奨します。ブラウザのデータ削除や、iOSのストレージ自動整理で消える可能性があります。
- 機種変更時は、旧端末で書き出したJSONを新端末で「バックアップから復元する」で読み込みます。

---

## Claude(artifact)版からデータを移す

1. Claude内のアプリで **設定** → **バックアップを書き出す** → JSONを保存
2. GitHub Pages版を開き、**設定** → **バックアップから復元する** → 保存したJSONを選択

## AI機能について

このバージョンでは AI 機能(AIコーチ・やり方解説・総括コメント)は **無効** です。

コードは残してあるので、将来 Anthropic API キーを使って復活させられます。

- `index.html` 内の `const AI_ENABLED = false;` を `true` に変更
- ただし **APIキーをフロントエンドに直書きするのは危険** です。実際に有効化する場合は、Vercel等でAPIキーを隠すプロキシ(サーバー関数)を1枚挟んでください。

なお、各種目の「▶ 動画」ボタン(YouTube でフォームを検索)は AI 不要で動作します。

## 更新の仕方

`index.html` を差し替えて再アップロードすれば更新されます。

Service Worker のキャッシュが効いているため、更新が反映されない場合は `sw.js` 内の

```js
const CACHE = "overload-v14";
```

のバージョン番号を上げてください(例: `overload-v15`)。
